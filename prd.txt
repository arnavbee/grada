PRODUCT OVERVIEW
What Problem Does It Solve?
D2C apparel brands selling on multiple Indian marketplaces face:
1. Catalog Chaos: 30-40 hours per collection creating catalogs manually, different format for each marketplace
2. PO Nightmare: 50+ purchase orders weekly, each taking 30-60 min to process manually
3. Invoice Hell: GST-compliant invoices and packing lists in marketplace-specific formats, all manual
4. No Unified View: Jumping between 5 different marketplace portals, no single source of truth
Total Cost: ₹15-25L annually in wasted labor + errors + opportunity cost
What Does Our MVP Do?
Module 1: Smart Catalog System
* Upload product images → AI extracts attributes (color, style, type, etc.)
* Upload tech pack PDF → OCR extracts measurements
* Auto-generate marketplace-ready catalogs
* One-click export to all 5 marketplaces in their specific formats
Module 2: PO Automation
* Receive POs via email/manual upload → AI parses all details
* Auto-validate against inventory
* Generate marketplace-specific confirmations
* Dashboard showing all POs in one place
Module 3: Invoice & Packing Automation
* Auto-generate GST-compliant invoices
* Create marketplace-specific packing lists
* Generate barcode labels
* One-click print/download
Module 4: Multi-Marketplace Dashboard
* Unified view of all orders across all marketplaces
* Real-time inventory sync
* Analytics and reporting
* Marketplace performance comparison

AI Integration Specifications
Image Analysis Pipeline
Input: Product image (JPG/PNG, max 10MB)
Process:
1. Image Upload
   → Store in S3/R2
   → Create image record in DB


2. Google Cloud Vision API Call
   → Labels detection (clothing type, attributes)
   → Dominant colors (RGB values)
   → Text detection (if any tags visible)
   → Safe search (filter inappropriate)
   
3. GPT-4 Vision API Call
   Prompt:
   "Analyze this apparel product image. Return JSON:
   {
     "garment_type": "specific type (e.g., Kurti, T-shirt, Jeans)",
     "style": "style classification (e.g., Casual, Formal, Ethnic)",
     "target_gender": "Men/Women/Unisex/Kids",
     "target_age_group": "age range",
     "pattern": "Solid/Striped/Floral/Printed/Embroidered/etc",
     "suggested_name": "compelling product name (50 chars max)",
     "description": "detailed product description (150-200 words)",
     "occasion": "suggested occasions to wear",
     "season": "suitable season(s)"
   }"


4. Combine Results
   → Merge Vision + GPT-4 results
   → Map colors to standard names
   → Calculate confidence score
   → Store in product.ai_analysis (jsonb)


5. Return to Frontend
   → Show AI suggestions
   → User can accept/edit


Output: Structured product data with confidence scores
Error Handling:
* Image too small/unclear → Return low confidence, flag for manual review
* API timeout → Retry 3x, then queue for later processing
* Inappropriate content → Reject image, notify user
Cost Optimization:
* Batch multiple images in single GPT-4 call (if API supports)
* Cache results for identical images (hash-based)
* Use cheaper Vision API for initial filtering, GPT-4 for detailed analysis

Tech Pack OCR Pipeline
Input: Tech pack PDF (max 5MB)
Process:
1. PDF Upload
   → Store in S3/R2
   → Create processing job


2. PDF to Images (if needed)
   → Convert each page to image
   → Or extract text if text-based PDF


3. OCR Processing
   → Use Tesseract OCR or Google Document AI
   → Extract all text from PDF
   
4. GPT-4 Text Analysis
   Prompt:
   "This is text extracted from an apparel tech pack. Find the measurements table and extract data. Return JSON:
   {
     "fabric": "fabric composition",
     "gsm": "fabric GSM if mentioned",
     "care_instructions": ["instruction 1", "instruction 2"],
     "measurements": {
       "XS": {"chest_cm": X, "waist_cm": Y, "length_cm": Z, ...},
       "S": {...},
       "M": {...},
       "L": {...},
       "XL": {...},
       "XXL": {...}
     }
   }
   If measurements are in inches, convert to cm. If any size is missing, mark as null."


5. Validation
   → Check if all expected sizes present
   → Flag unrealistic measurements (e.g., chest > 200cm)
   → Calculate size progression (each size should increase logically)


6. Return Results
   → Display in editable table
   → Show validation warnings


Output: Structured measurements data
Error Handling:
* Poor quality scan → Low confidence, suggest re-upload
* No measurements found → Flag for manual entry
* Partial data → Use what's available, flag gaps

PO Parsing Pipeline
Input: PO file (PDF or Excel) + marketplace hint
Process:
1. File Type Detection
   → If Excel: Use pandas to read
   → If PDF: Extract text using PyPDF2 or pdfplumber


2. Marketplace Detection
   → Look for marketplace name/logo in text
   → Match email domain if from email
   → Check PO number format (each marketplace has pattern)
   → Use hints from user if provided


3. GPT-4 Parsing
   For Excel:
   Prompt:
   "This is a purchase order from {marketplace}. Extract data and return JSON:
   {
     "po_number": "PO number",
     "po_date": "YYYY-MM-DD",
     "delivery_deadline": "YYYY-MM-DD",
     "warehouse_address": "full address",
     "buyer_info": {"name": "", "email": "", "phone": ""},
     "line_items": [
       {"sku": "", "description": "", "color": "", "size": "", "quantity": X, "unit_price": Y, "total": Z}
     ],
     "subtotal": X,
     "tax": Y,
     "grand_total": Z,
     "special_instructions": ["instruction 1", ...]
   }"
   
   For PDF (similar but adapted for text format)


4. Validation
   → Verify all required fields extracted
   → Check if SKUs exist in client's catalog
   → Validate math (line totals, grand total)
   → Flag discrepancies


5. Human Review Queue
   → If confidence < 90%, flag for review
   → If SKUs not found, flag
   → If totals don't match, flag


Output: Structured PO data with validation status
Marketplace-Specific Logic:
* Myntra: Excel format, specific column names
* Amazon: May come via API or Excel, ASN required
* Flipkart: Excel, QC requirements
* Ajio: PDF usually, specific format
* Nykaa: Excel, beauty-specific fields
